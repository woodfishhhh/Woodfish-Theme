# 更新日志

本文件记录了 Woodfish Theme 项目的所有重要更改。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
并且本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

> 说明：
> 从 4.x 开始，Woodfish Theme 进入 integrated runtime 时代，主路径是内置 workbench 注入。
> 旧版本 (3.x 及更早) 的记录保留历史事实，其中出现的 Custom CSS / 外部 Loader 相关描述仅代表当时实现。

## [5.1.4-beta] - 2026-04-10

### 修复

- 🌈 **彩色光标流光恢复** - 去掉 `bp-animation` 关键帧中 `background-position` 的无效 `!important`，让彩色光标重新完整流过赤橙黄绿青蓝紫，而不是只停在开头的红橙段。
- 🧪 **光标动画回归保护** - 新增运行时 payload 测试，断言 `bp-animation` 关键帧里的 `background-position` 保持可动画，避免未来再次打包出“有渐变但不流动”的光标。

## [5.1.3] - 2026-04-08

### 修复

- 🛠️ **升级后备份路径回退** - 当持久化状态里仍保留旧 VS Code 版本目录下的 `workbench.html.woodfish-backup` 时，运行时现在会自动回退到当前 `workbench.html` 所在目录的新备份路径，避免启用或修复注入时触发 `ENOENT`。
- 🧪 **备份路径回归测试** - 新增针对“旧 `backupPath` 已失效”场景的服务级测试，确保未来升级 VS Code 后不会再次把备份写到不存在的旧版本目录。

## [5.1.2] - 2026-04-07

### 更改

- 🧹 **仓库遗物清理** - 删除已确认无引用的旧入口 shim、旧 Bash 发布脚本、无用版本检查代码和两份过时图片说明，仓库主线进一步收口到当前 TypeScript + integrated runtime 结构。
- 🧪 **清理回归测试补强** - 将占位测试替换为仓库清理保护测试，并补充对旧架构说明回潮的断言，避免旧入口、旧发布流或旧文案再次混回主线。
- 📝 **开发文档对齐** - `docs/CONTRIBUTING.md` 与 `.github/copilot-instructions.md` 改写为当前 5.x integrated runtime 架构，不再指向旧的 Loader / `src/lib/*` / 已删除资源链路。

## [5.1.1] - 2026-04-07

### 修复

- 📝 **扩展描述修正** - Marketplace 展示文案改为当前稳定版的一体化 Woodfish 主题表述，不再保留“回档到稳定版本”的过时描述。
- 🧭 **状态栏刷新修正** - `Reload Window` 提示改为非阻塞链路，彻底停用或移除注入后，状态栏会立即按真实 payload 检测结果切换为 `off`。

## [5.1.0] - 2026-04-07

### 更改

- 🧹 **设置面板瘦身** - 删除 `woodfishTheme.runtime.enabled`、`woodfishTheme.runtime.autoSwitchTheme`、`woodfishTheme.runtime.reapplyOnStartup` 与 `woodfishTheme.syntaxGradient.preset`，只保留真正会产生视觉变化的设置项。
- 🧠 **运行态真值化** - 状态栏、菜单与命令流程不再依赖伪开关，而是按当前主题与 `workbench.html` 里的真实 payload 检测结果显示 `on / paused / off`。
- 📝 **设置说明增强** - 保留项说明补充了单位、效果、原理、例子和为什么要提供该设置。
- ♻️ **旧版 Woodfish payload 接管** - 启用或修复时会识别并接管已知旧版 Woodfish 注入，再写入新的一体化 payload；未知第三方注入不会被自动改写。
- 📚 **文档同步** - README、排障文档和示例配置统一切换到新的设置模型。

## [5.0.2] - 2026-04-07

### 更改

- 🧭 **状态栏菜单补全指令** - 点击右下角 `Woodfish` 状态栏后，菜单现在会直接显示全部主要指令，包括开启/关闭主题、彩色字体、发光字体、彩色光标、修复注入、彻底停用以及 `Reload Window`。

## [5.0.1] - 2026-04-07

## [5.0.0] - 2026-04-06

### 更改

- 🚀 **一体化 runtime 默认启用** - 首次切换到 `Woodfish Dark` 后即可走内置注入主路径，不再要求额外开启第三方 CSS Loader。
- 🌈 **彩色光标首开体验收口** - 保持旧命令 id 兼容，但对外文案统一为一体化 runtime 语义。
- ✨ **首装 glow 默认链路修复** - 配置读取改为完整键路径，fresh 安装时彩色字体 / 发光字体 / 彩色光标可一起按默认值生效。
- 🧹 **遗留兼容链瘦身** - 删除旧 `customCss` 服务链、依赖提示和相关调试脚本，仓库主线只保留 integrated runtime。
- 📝 **发布文案同步** - README、排障文档与发布检查说明统一改为 runtime 主线表述。
- 📦 **发布包收口** - VSIX 过滤掉 `plan.md` 和开发配置文件，发布脚本改为强制执行 compile/lint/test/预检后再打包。

## [4.0.0] - 2026-02-21

### 更改

- 🧱 **架构完全重构** - 扩展入口改为编排层，命令、配置、服务、UI 全部模块化拆分（`commands/`、`config/`、`services/`、`ui/`）。
- 🧩 **CSS 注入服务化** - 注入、清理、路径选择逻辑拆分为独立服务模块，降低耦合并提升可维护性。
- 🎛️ **状态同步重构** - 新增集中式 feature state 控制器，统一管理配置读取、状态栏更新和命令执行状态。
- 🧪 **迁移准备阶段** - 为后续 integrated runtime 主线铺设运行时注入、配置同步和 workbench 修复能力。

### 移除

- ❌ **透明/毛玻璃 UI 功能** - 移除 `toggleGlassEffect` 命令、`woodfishTheme.enableGlassEffect` 配置项和对应样式入口。
- ❌ **玻璃样式构建入口** - 删除 `index-with-glass.css`，并从组合样式 `index-all.css` 中移除透明样式导入。
- ❌ **旧版 lib 结构** - 删除旧 `src/lib/*` 实现，统一迁移至新的分层目录。

### 修复

- 🧹 **历史残留清理兼容性** - 保留对旧 `transparent-ui.css` 导入残留的清理匹配，确保升级后自动清理旧配置。
- ✅ **构建与测试验证** - TypeScript 编译通过，Jest 单进程测试通过。

## [3.5.0] - 2025-12-25

### 更改

- 🏗️ **TypeScript 重构** - 扩展逻辑迁移为 `src/` → `out/` 的 TS 构建产物，并以编译结果作为运行入口
- 📁 **主题结构调整** - 主题资源统一整理到 `themes/Bearded Theme/`，并同步修正扩展贡献点路径
- 🧰 **工具链调整** - 移除 ESLint，统一使用 Prettier 进行格式化与校验
- 📝 **文档与注释** - 同步更新中英文 README，并补全核心模块的工程级 TSDoc

### 修复

- ⚡ **命令激活可靠性** - 为所有命令补齐 `onCommand:*` 激活事件，避免仅依赖启动完成后激活
- 🧹 **CSS 注入清理更安全** - 收紧 imports 匹配规则，降低误删用户其他自定义 CSS 注入项的风险
- 🔗 **URI 解析更稳健** - 清理 imports 时使用 `vscode.Uri` 解析 `file:///` 路径，避免因编码差异误判
- 📦 **打包兼容性** - 对齐 `@types/vscode` 与 `engines.vscode`，确保 `vsce package` 可通过校验

## 版本说明

### 版本号格式

- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正

### 更新类型说明

- **新增**：新功能
- **更改**：对现有功能的变更
- **弃用**：不再推荐使用的功能
- **移除**：已删除的功能
- **修复**：问题修复
- **安全**：安全相关的修复
